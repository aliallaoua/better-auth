# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Tooling

- **Runtime/package manager**: Bun (not npm/pnpm/yarn)
- **Linter/formatter**: Biome via Ultracite preset — run `bun run lint:fix` to auto-fix
- **Test runner**: Vitest
- **Database**: Drizzle ORM with Neon PostgreSQL
- **Shadcn**: install components with `bunx shadcn@latest add <component>`

## Commands

- `bun run dev` — dev server (port 3000)
- `bun run build` — production build
- `bun run test` — run all tests
- `bunx vitest run src/path/to/test.ts` — run a single test
- `bun run check` — lint + format check
- `bun run lint:fix` — auto-fix lint/format issues
- `bun run db:push` — push schema changes to database
- `bun run db:generate` — generate migrations
- `bun run db:migrate` — run migrations

## Non-obvious Conventions

- `src/routeTree.gen.ts` is auto-generated by TanStack Router — never edit it
- `src/components/ui/` is shadcn-managed and excluded from Biome linting — don't manually modify these files
- Environment variables are Zod-validated: server env in `src/config/server-env.ts`, client env in `src/config/client-env.ts`. Add new env vars to the schema, not just `process.env`
- The database schema is in `src/db/schema.ts`
- Use `import type` with separated type style (enforced by Biome)
- React Compiler is enabled — avoid patterns that break React Compiler rules
- RTL support is enabled in shadcn config
- Formatting uses tabs, double quotes, ES5 trailing commas
